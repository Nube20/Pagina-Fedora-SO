Antecedentes

Fedora comenzó su andadura en 2003 como un proyecto derivado de Red Hat Linux, una de las primeras distribuciones de Linux y líder en el mercado de servidores. Fedora fue creada como un proyecto comunitario para explorar nuevas ideas y tecnologías en el mundo de Linux. El nombre de Fedora deriva de Fedora Linux, un proyecto creado por voluntarios que proveía software adicional a la distribución Red Hat Linux, y del característico sombrero Fedora usado en el logotipo de la distribución comercial.

Características técnicas y generales

Comunidad

Cuenta con una comunidad de desarrolladores y usuarios activa y comprometida, que colabora en la mejora y evolución del sistema operativo.

Seguridad

Se enfoca en la seguridad, implementando políticas de seguridad estrictas y actualizaciones constantes.

Estabilidad

Aunque es una distribución de vanguardia, Fedora trabaja para mantener la estabilidad y la confiabilidad en cada lanzamiento.

Flexibilidad

Es altamente personalizable y se adapta a las necesidades de diferentes usuarios y casos de uso.

Su navegador por defecto es Firefox desde su versión Fedora Core 3 y superior.

Fedora soporta las arquitecturas x86, x86-64 y PowerPC.

Soporta redes instaladas sobre HTTP, FTP y NFS.

El entorno de escritorio por defecto es GNOME, e incluye KDE en algunas versiones.

Gestor de arranque GNU GRUB

Permite a los usuarios tener múltiples sistemas operativos en la misma computadora y permite también elegir cuál ejecutar cuando la misma se inicia.

Es dinámicamente configurable; abre su configuración al inicio, permitiendo en tiempo de inicio cambios como seleccionar diferentes kernels. GRUB también provee una simple interfaz de comandos, que permite a los usuarios escribir nuevas secuencias de buteo.

Es altamente portable. Soporta múltiples formatos de ejecutables y también soporta sistemas operativos que no permiten multibuteo como Windows o OS/2.

Sistemas de archivos que administra

Sistema de archivos con registro por diario (Journaling). Fue creado para superar las limitaciones del sistema de archivos EXT2. Mejora la confiabilidad y elimina la necesidad de verificar el sistema de archivos después de un cierre no limpio.

Uno de los sistemas de archivos más nuevos que hay disponibles en Linux. La meta de Btrfs es implementar algunas características importantes que faltan en los sistemas de archivos en Linux, tales como pooling, snapshots y checksums, entre otras.

Aplicaciones del sistema en la industria

1Servidores web

Fedora es una opción popular para la creación y administración de servidores web, gracias a su capacidad para ejecutar aplicaciones web y servidores de bases de datos como Apache, Nginx, MySQL y PostgreSQL.

2Redes de almacenamiento

Fedora es una plataforma sólida para la creación de sistemas de almacenamiento en red (NAS) y sistemas de almacenamiento en la nube (SAN) gracias a su compatibilidad con tecnologías como el sistema de archivos Btrfs y los servicios de red como NFS y Samba.

3Virtualización

Fedora incluye una gran cantidad de herramientas para la virtualización de sistemas, lo que la convierte en una opción popular para la creación de entornos de virtualización y para la administración de servidores virtuales. Entre las tecnologías de virtualización que se pueden utilizar en Fedora se incluyen QEMU, KVM, VirtualBox y Docker.

4Automatización y administración

Fedora incluye una amplia variedad de herramientas de automatización y administración de sistemas, lo que la convierte en una opción popular para la creación y el mantenimiento de sistemas complejos. Entre las herramientas de automatización que se pueden utilizar en Fedora se incluyen Puppet, Ansible y Salt.

5Desarrollo de software

Fedora es una plataforma sólida para el desarrollo de software, gracias a su compatibilidad con una amplia variedad de lenguajes de programación y herramientas de desarrollo. Algunas de las herramientas de desarrollo que se pueden utilizar en Fedora incluyen GCC, Python, Ruby, Perl y Java.

Interfaces

En Fedora, una interfaz de red es una conexión de hardware que permite que un dispositivo se comunique con otros dispositivos en una red. En términos simples, es el medio a través del cual tu computadora se conecta a Internet o a una red local.

Las interfaces de red pueden ser físicas o virtuales. Las interfaces físicas son los adaptadores de red reales, como tarjetas Ethernet, adaptadores Wi-Fi, adaptadores Bluetooth, etc. Las interfaces virtuales, por otro lado, son interfaces lógicas que se crean para permitir que un solo adaptador de red real se use para múltiples propósitos.

En Fedora, puedes administrar y configurar tus interfaces de red utilizando la herramienta de línea de comandos "nmcli" o la interfaz gráfica de usuario "NetworkManager". Puedes configurar tu interfaz de red para que obtenga una dirección IP automáticamente a través del protocolo DHCP, o puedes configurarla manualmente con una dirección IP estática. También puedes configurar otras opciones de red, como el DNS, la puerta de enlace predeterminada y la máscara de red, para tu interfaz de red.

Es importante conocer tus interfaces de red en Fedora, ya que esto te permite conectarte y comunicarte con otras computadoras en una red.
